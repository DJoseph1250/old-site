Ruby Cartography

3/1/2015

  In case the title wasn't enough of a hint, this blog post will be dedicated to mapping. Not the Amerigo Vespucci kind. The enumerable. Confused? Well hopefully by the time you're done reading this you won't be. Let's start with explaining enumerables before we get into map.

 Enumerables are ways to search, sort and alter arrays, hashes and the values inside of them. What if you wanted to change the properties of all the values in an array? If it's a short array you could just call each individual one. It's inefficient, but it would work. However, what if it's an array with hundreds of values? Then you should use a loop. But what if there was a simpler way? A way to influence each entrance, or the ones that you just need, on only one line? Here's where enumerables come to save the day. Enumerables allow one to iterate over an entire hash or array. What does it look like? Well here's an example using map:

array_name.map {|v| return v}

 You've probably noticed the curly brackets and their content. The part inside the lines represents the objects or values inside of the array. The part after that is the block. It helps the enumerable know what exactly it is that you want to do and what you want to do it to. Not every enumerable requires a block afterwards to do something, but they all can work with one. But I still haven't told you what map does.

  Map is a non-destructive enumerable. This means that it doesn't change the original array it has been called on. It creates an exact copy of an array. Hence the name map. Then you can do whatever you want to that array, without altering the original. However, there is a destructive version of map known as map!. The exclamation mark at the end tells ruby to iterate over the original. For example, if array = [1,2,3,4,5] then array.map! {|i| i*2} will change the values in array to double what they are or array = [2,4,6,8,10]. Whereas with ordinary map it will create a copy. So if you wanted to make a copy and iterate over that you'd do this:

array = [1,2,3,4,5]
array_clone = array.map {|i| i*2}

And then the value of array_clone would change to be the values of array doubled, while the values of array would stay the same. So that's how you use map and if you don't know, now you know.
