Visual Editor

9/7/2015

 Last week I was given a task that required me changing one line in a text file, to which my response was, "I'm not opening a file in sublime just to change one line. There must be a way to edit text files in the command line since I'm already here". Horrifically lazy, I know, I'll survive. But it was also a question I'd had for a while and never got around to learning the answer. I knew how to create, read, and delete files, but not how to edit them. Now was the perfect time. So like many before me who thirsted for knowledge, I went to the omniscient Google for the answer.

  vi filename

 That's it. Just type into the console 'vi' and then the filename. The 'vi' stands for 'Visual Editor'. "But Djavan, shouldn't it be 've' then"? It should, but the choice wasn't mine. Instead they decided to choose the first two letters in 'visual'. If the filename exists it will open it in the command line. If it doesn't, it will create it and then open it. However, you can't just start typing when the file opens in the command line. The command line Visual Editor has it's own commands that need to be used.

 There are quite a lot, that either do nearly the exact same thing or aren't that useful at all, so I'll only go over some very basic ones. I doubt any of us will be using Visual Editor to edit 1000 line programs and it shouldn't be used for that. With text editors around, it's only good for creating and editing small files and that's mainly what I do with it, if I even use it at all. So these commands should suffice:

  • x
    Used for deleting characters while not in edit mode.
  • dd
    Deletes the entrie line the cursor is currently on.
  • i
    Used to enter edit mode. From there you can add and delete characters normally. Press esc to exit edit mode.
  • u
    Used to undo any action that was just completed. Cannot be used to undo multiple actions like command-z(ctrl-zfor windows).
  • :x
    Unlike the others you must press enter/return after inputting this command or else it won't execute. It's used to exit a file and save any changes made. Cannot be used while in edit mode.
  • :q
    Just like :x, enter/return must be pressed afterwards. It is used to ezit a file with either no changes or already saved changes. If you attempt to exit a file with saved changes, without saving them, it will tell you to use the next command in this list.
  • :q!
    Used to forcefully exit a file with unsaved changes.

 And that's all that you will probably ever need. If you want to know more there are useful cheat sheets out there. Here's one.This also isn't the only way to edit text files in the console. There is another way that I know of, known as 'emacs'. But I'm not too familiar with it. If you want to know more about it, I'm sadly not your guy. But thankfully google exists.
