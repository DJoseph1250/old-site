Arrays & Hashes

2/22/2105

  Arrays and Hashes. Similar yet distinctly different. In this blog post I will attempt to teach you the difference and the appropriate situation in which to use them. Let's start with Arrays.

  In Ruby, Arrays are lists. But unlike most lists they start counting from 0 instead of 1. The first entrance in an Array is actually the 0th (yes I know that's not a real term). This can be a bit confusing at first, but over time it should sink in. There are 2 ways to start an Array:

Array_name = Array.new

and

Array_name = [value,value,value...]

 I prefer to use the latter. It seems simpler to me to start the Array, name the Array and input values at the same time. When using the former you MUST capitalize when typing Array.new. If you want to change a specific entrance in an Array, you merely have to remember which number on the list it is (remember we start counting from 0!) and do this:

Array_name[n] = Input

where n equals the position in the array (0, 1, 2, 3...).

  Now I've been talking about Arrays but I've yet to mention what you can put in them. Well, strings and numbers. Now if you're asking if that's all, how about Arrays? That's right, you can put Arrays inside of Arrays. Arrayception.



If you're having trouble visualizing that, here you go:

Array_name = [[0],[1],[2]]

 That's what it looks like. Hopefully your mind hasn't been completely blown yet because that's not all. You can also put Hashes inside of Arrays too! But we don't even know what a Hash looks like yet, or even what it is.

 If an Array is an ordered list, a Hash is an unordered one. Where the values in an Array are assigned to numbers signifying their place in the list, the values in a Hash are assigned to a key of your choosing. If you're slightly confused, don't worry I'm about to give you an example. Hashes are started the exact same way Arrays are, except instead of using [] you use {}. So it looks like this:

Hash_name = Hash.new

or

Hash_name = {key => value, key => value, key => value...}

 Keys can be strings although I prefer to use symbols. Instead of "key" it would be :key. Now you might have wondered already "Can keys also be numbers?". Well yes but usually that's what arrays are for. Hashes are for when you want to associate one thing with another. For example, birthdays with names or a color with an emotion. If I wanted to keep track of the type of car my friends and family drives I'd use a hash:

Array_name = {'Sue' => 'Coupe', 'Bob' => 'Truck', 'Tom' => 'Sedan', 'Mary' => 'Hybrid'}

 The keys are the names and the values are the cars. Just like you can put Arrays and Hashes into Arrays, you can also put them both into Hashes. You could have a Hash inside of an Array inside of a Hash. This might not sound too interesting, but the possibilities are very interesting. I hope this post helped you and if not, then there's an old Confucian saying, Google knows all!
